# 專題製作學習歷程牆 · 設置指南

給老師的零程式碼部署手冊。全程跟著做，預計 **30 ~ 45 分鐘** 可以上線。

---

## 架構速覽

```
[學生] → 填 Google 表單 → 檔案進 Google Drive 資料夾
                         → 文字紀錄進 Google 試算表
                                   ↓
                  [GitHub Pages 網站] 每 30 秒自動抓取
                                   ↓
                    [上方動態牆 + 下方 28 格學生區]
                                   ↕
   [選用] emoji 反應 / 留言 ←→ Google Apps Script ←→ 同一份試算表的 Reactions / Comments 分頁
```

你需要三個東西：一個 **Google 帳號**、一個 **GitHub 帳號**，以及這個資料夾裡的六個檔案。核心部署（前三部分）只用 Google 表單 + GitHub Pages，完全不寫程式；互動功能（第四部分）才需要貼一次 Apps Script。

---

## 第一部分：Google 端設置（約 20 分鐘）

### 步驟 1 ── 建立 Google 雲端資料夾

1. 打開 [drive.google.com](https://drive.google.com)
2. 左上「新增」→「新資料夾」，取名：`學生專題產出`
3. 進入這個資料夾，點右鍵 →「共用」→ 把下方權限從「受限制」改為「知道連結的任何人 · 檢視者」
4. 這個資料夾將會自動收集學生上傳的所有檔案

### 步驟 2 ── 建立 Google 表單

1. 打開 [forms.google.com](https://forms.google.com)，點「建立空白表單」
2. 把表單標題改為：`上傳我的專題產出`
3. 依序新增下列 **7 個欄位**（重要：欄位名稱要和下列完全一致，因為程式會依名稱抓資料）：

| # | 欄位名稱 | 類型 | 設定 |
|---|---|---|---|
| 1 | 班級 | 單選 | 選項：`A 班`、`B 班`，設為**必填** |
| 2 | 學生姓名 | 下拉式選單 | 列入 28 位同學，設為**必填** |
| 3 | 產出類型 | 單選 | 選項：`圖片`、`影片`、`文件`、`連結`、`大頭貼`，設為**必填** |
| 4 | 標題 | 簡短回答 | 設為**必填** |
| 5 | 說明 | 段落 | 非必填 |
| 6 | 檔案上傳 | 檔案上傳 | 非必填（開啟後會跳出權限確認，點同意） |
| 7 | 外部連結 | 簡短回答 | 非必填，用來貼 YouTube / Figma / GitHub 等 |

4. **設定檔案上傳的目的地**：
   - 點第 6 題旁邊的齒輪（或在題目中點「變更資料夾」）
   - 選擇步驟 1 建立的「學生專題產出」資料夾

> 💡 **關於「班級」欄位**：網站會根據這個欄位提供 `全部 / A 班 / B 班` 三種篩選視角、顯示班級色彩標籤、並把 28 格同學牆分成兩區。若你不希望分班，把這個欄位從表單刪掉即可，網站會自動忽略（但記得 `config.js` 的 `classes` 也要一併清空）。

5. 點右上「傳送」→ 切換到「🔗 連結」標籤 → 勾選「縮短網址」→ 複製這個網址（**這是學生上傳用的網址**，等下要貼到 config.js）

### 步驟 3 ── 綁定試算表並取得 Sheet ID

1. 回到表單編輯頁，點「回覆」標籤
2. 點右上綠色的「📊 連結至試算表」→「建立新試算表」→ 確定
3. 試算表會自動開啟，從網址列複製 Sheet ID：
   ```
   https://docs.google.com/spreadsheets/d/【這一長串就是 Sheet ID】/edit
   ```
4. **把試算表設為「知道連結的任何人 · 檢視者」**：
   - 右上「共用」→ 把權限改為「知道連結的任何人 · 檢視者」
   - 這一步必做！否則網頁抓不到資料

### 步驟 4 ── 測試一次

1. 用剛剛複製的表單網址，自己試填一則
2. 回到試算表，確認那一列有出現
3. 回到 Drive 資料夾，確認檔案有進去

如果這三件事都成立，Google 端就完成了 ✅

### 步驟 5（選用）── 建立「老師示範用」第二張表單

如果你擔心學生在一般表單中誤選「Chibi（X 班示範）」搗亂，可以另外開一張老師專用的表單，兩張表單會寫進**同一份試算表**的不同分頁，網站會自動同時讀取並合併。

1. 到 [forms.google.com](https://forms.google.com) 新增一張表單，命名為：`【老師示範】專題產出上傳`
2. 把**欄位結構**做得跟學生表單一模一樣（一樣的 7 個欄位、一樣的名稱與順序），只差兩個地方：
   - 第 1 題「班級」下拉**只放一個選項**：`X 班`
   - 第 2 題「學生姓名」下拉**只放一個選項**：`Chibi`
3. 第 6 題「檔案上傳」目的地，指向**同一個** `學生專題產出` 資料夾（或你另外準備的示範資料夾也可以）
4. 到「回覆」標籤 → 點綠色「連結至試算表」→ 選「**選取現有試算表**」→ 選你第一張表單綁定的同一份試算表 → 確定
5. 試算表會**自動新增一個分頁**，名稱類似「表單回應 2」
6. 打開那個新分頁，從網址列抓它的 `gid`：
   ```
   https://docs.google.com/spreadsheets/d/.../edit#gid=【這串就是新 gid】
   ```
7. 編輯 `config.js` 裡的 `sheetGids` 陣列，把新的 gid 加進去：
   ```js
   sheetGids: [
     "230117923",  // 學生表單
     "新 gid 貼這裡",  // 老師示範表單
   ],
   ```
8. 重新上傳到 GitHub，網站就會同時讀兩個分頁

**老師示範時的流程**：打開這張老師表單 → 填 Chibi 的資料 → 送出 → 網站 30 秒內會在最上方顯示「X 班（示範）」的新動態，學生看得到但無法冒用。

## 第二部分：設定你的網頁（約 5 分鐘）

用任何文字編輯器打開 `config.js`（可以直接用 macOS 內建的「文字編輯」或 VS Code），修改下列三個地方：

```js
const CONFIG = {
  siteTitle: "專題製作學習歷程牆",      // 改成你的班級標題
  siteSubtitle: "2026 春季班 · ...",   // 副標題

  sheetId: "貼上步驟 3 複製的 Sheet ID",   // ← 改這裡
  sheetName: "表單回應 1",                 // 一般不用改

  formUrl: "貼上步驟 2 最後複製的表單網址", // ← 改這裡

  classes: [
    { id: "A", label: "A 班", color: "#0ea5e9" }, // 想換顏色改 color 即可
    { id: "B", label: "B 班", color: "#f59e0b" },
  ],

  students: [
    { name: "王小明", class: "A" },  // ← 改成你班上的真名與班別
    { name: "陳美麗", class: "A" },
    ...
    { name: "藍思穎", class: "B" },
  ],
};
```

**注意事項**：
- `students` 裡每個 `name` 必須跟你在 Google 表單「學生姓名」下拉選單裡寫的**完全一致**（一個字都不能差），否則網頁分組會失敗
- `students` 裡每個 `class` 必須對應 `classes` 裡的 `id`（例如 `"A"` 或 `"B"`）
- `classes[].label` 要跟表單「班級」欄位的選項**完全一致**（例如都是「A 班」中間有一個半形空格）

### 在本機先看看效果（選用）

1. 打開「終端機」，執行：
   ```bash
   cd ~/Desktop/student-showcase
   python3 -m http.server 8000
   ```
2. 瀏覽器打開 <http://localhost:8000>
3. 看看資料有沒有出現、版面是否 OK

Ctrl+C 可停止本機伺服器。

## 第三部分：部署到 GitHub Pages（約 10 分鐘）

### 步驟 1 ── 註冊 GitHub 帳號

若還沒有，到 [github.com](https://github.com) 註冊一個免費帳號。

### 步驟 2 ── 建立新 Repository

1. 登入後點右上「+」→「New repository」
2. Repository name 填：`student-showcase`（或你想要的名字）
3. 設為 **Public**（GitHub Pages 免費版需要 Public）
4. **不要**勾選「Add a README file」
5. 點「Create repository」

### 步驟 3 ── 上傳檔案（最簡單的網頁方式）

1. 在剛建好的 repo 頁面，點「uploading an existing file」連結
2. 把 `~/Desktop/student-showcase/` 資料夾裡的 **所有檔案**（`index.html`、`style.css`、`app.js`、`config.js`、`README.md`）拖曳進去
3. 下方點「Commit changes」

### 步驟 4 ── 啟用 GitHub Pages

1. 進入 repo 頁面，點上方「Settings」
2. 左側選單點「Pages」
3. 在「Source」選「Deploy from a branch」
4. Branch 選 `main`，資料夾選 `/ (root)`，點「Save」
5. 等 1~2 分鐘，上方會出現你的網址，類似：
   ```
   https://你的帳號.github.io/student-showcase/
   ```
6. 打開就能看到你的學習歷程牆了 🎉

### 以後要修改設定怎麼辦？

例如新增一位學生、改標題：
1. 在 repo 裡點 `config.js` → 右上鉛筆圖示
2. 直接在網頁上修改
3. 下方「Commit changes」
4. 等 1 分鐘，網站就會更新

## 第四部分（選用）：啟用 Emoji 反應與留言（約 15 分鐘）

這一段讓學生能在每件作品下方按 👍 / ❤️ / 🔥 / 😂 / 👀，以及留下文字留言。不做這段，網站照樣正常運作，就是沒有互動功能。

### 運作原理

- 同一份 Google 試算表會自動多兩個分頁：`Reactions`（反應）與 `Comments`（留言）
- 背後是一段綁定在試算表上的 **Google Apps Script**，負責讀寫這兩張表
- 學生不需要登入 Google 帳號 ── 第一次進網站會跳出「我是...」的選擇器，從你班上的名單選一個名字，存在瀏覽器裡
- 每 10 秒自動刷新，感覺近即時

### 步驟 1 ── 部署 Apps Script

1. 打開你綁定表單的那份 Google 試算表
2. 上方點「擴充功能」→「Apps Script」，會開啟一個新的腳本編輯器
3. 把編輯器裡預設的 `function myFunction() {}` 全部刪掉
4. 打開資料夾裡的 `apps-script.gs`，把**整個檔案**內容複製貼到 Apps Script 編輯器
5. 按上方磁碟圖示存檔（檔名隨意，例如「showcase-backend」）
6. 點右上藍色「部署」→「新增部署作業」
7. 左上齒輪點一下，選「網頁應用程式」
8. 填：
   - 說明：隨意，例如「v1」
   - 執行身分：**我**（你自己）
   - 存取權：**任何人**（一定要這個；不是「任何擁有 Google 帳戶的使用者」）
9. 點「部署」。第一次會跳出授權視窗：
   - 點「授權存取權」
   - 選你的 Google 帳號
   - 會看到「Google 尚未驗證這個應用程式」警告 → 點「進階」→「前往 showcase-backend（不安全）」→「允許」
   - 這是因為腳本是你自己寫的、沒送 Google 審核，是正常畫面
10. 完成後會拿到一個「網頁應用程式網址」，長得像：
    ```
    https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxxxxxxxxx/exec
    ```
    把這整串複製起來

### 步驟 2 ── 把網址貼到 config.js

打開 `config.js`，把剛剛那串網址貼到 `appsScriptUrl`：

```js
appsScriptUrl: "https://script.google.com/macros/s/AKfycbx.../exec",
```

### 步驟 3 ── 驗證

1. 重新上傳 `config.js` 到 GitHub（或本機測試）
2. 打開網站，右上會多一個「我是：選擇身分 ▾」的按鈕
3. 點它，從名單選一個名字
4. 點任何一位學生的格子 → 在彈窗裡按一個 emoji → 應該立刻看到數字 +1
5. 試著留一則留言 → 送出後立刻出現
6. 回到 Google 試算表，你會看到**自動新增了 `Reactions` 和 `Comments` 兩個分頁**，剛剛的動作都被記錄下來

### 老師怎麼管理？

- 所有反應與留言都在試算表裡，**刪掉某一列就等於把那則刪除**（網站最多 10 秒後會同步）
- `Comments` 分頁的 `text` 欄位是學生的留言文字，可以直接編輯修正錯字
- 想看某位同學收到哪些留言，用試算表的篩選功能過濾 `entryId` 欄位即可

### 更新 Apps Script 之後怎麼辦？

如果之後你修改 `apps-script.gs` 的程式：
1. 複製新版本貼回 Apps Script 編輯器、存檔
2. 點「部署」→「管理部署作業」
3. 點右上鉛筆圖示 → 版本選「新版本」→「部署」
4. 網址不變，不用改 `config.js`

### 常見狀況

**Q：點了 emoji 沒反應？**
- 打開瀏覽器開發者工具（F12）→ Console，看有沒有紅色錯誤訊息
- 如果看到 CORS 錯誤，代表 Apps Script 部署時「存取權」沒選「任何人」
- 如果看到 404，檢查 `appsScriptUrl` 是不是完整的、結尾有 `/exec`

**Q：學生名字被別人冒用怎麼辦？**
- 這是弱實名的極限。試算表裡可以看到所有留言的時間戳記與 `userId`，對照課堂情境通常看得出來
- 最簡單：上課當面講清楚規範、出事該列直接刪掉
- 想更硬一點可以改成「認領碼」機制（每位學生一個一次性密碼，綁定裝置），這是後續可加的功能
- 真的要完全防冒用就要做 Google OAuth 登入，那是另一個大工程

**Q：想暫時關掉反應和留言（例如考試週）？**
- 把 `config.js` 裡的 `appsScriptUrl` 改成空字串 `""`
- 網站會自動隱藏身分選擇器、emoji 列和留言區
- 試算表裡歷史資料還留著，改回去就又出現了

### 訪客模式（給老師 / 家長 / 其他來賓）

預設是**開啟**的。身分選擇視窗底部會多一個「自填暱稱」區塊，可以自己打名字，並選擇是「訪客」「老師」或「家長」。留言會自動加上對應的徽章（例如「👩‍🏫 老師」「👨‍👩‍👦 家長」），方便你在試算表裡一眼分辨。

- 訪客的暱稱**不能跟學生同名**，系統會擋下來
- 訪客的紀錄在 Google Sheet 的 `Comments` 分頁，有一欄 `role` 區分身分（student / guest / teacher / parent）
- 想關閉（例如只開放給學生）把 `config.js` 的 `guestModeEnabled` 改成 `false` 即可
- 也可以在 `guestRoles` 陣列裡增減角色選項

### 學生大頭貼（選用）

每位學生可以有一張代表用的大頭貼，取代學生格子的縮圖。預設是**開啟**的。

**老師要做的**（一次設定）：
- 確認表單第 3 題「產出類型」的選項裡有一個 `大頭貼`（或任何你在 `config.js → avatarType` 設定的字串）
- 如果之前的表單沒加，現在進去表單編輯頁補上那個選項即可

**學生要做的**：
- 用**同一張**上傳表單
- 類型選 `大頭貼`，檔案欄上傳一張圖（標題、說明可以隨便填或留空）
- 想換就再填一次，系統會用**時間最新**那張
- 大頭貼資料會跟作品一起放在 Google 試算表，但**不會進動態牆、不算進「N 則產出」**

**想關閉或改名稱**：
- 把 `config.js` 裡的 `avatarType` 改成空字串 `""` 就完全關閉
- 想改叫「頭像」「個人圖」等，把字串改掉就好（表單那邊的選項要同步改）

## 常見問題

**Q：網頁顯示「載入失敗」怎麼辦？**
- 檢查 `config.js` 裡的 `sheetId` 是否正確
- 檢查試算表權限是否為「知道連結的任何人 · 檢視者」
- 打開瀏覽器的開發者工具（F12）→ Console，看錯誤訊息

**Q：有學生上傳了，但網頁沒顯示他的產出？**
- 檢查 `config.js` 的 `students` 陣列裡，他的名字有沒有打對
- 或等 30 秒，網站會自動刷新

**Q：圖片顯示不出來？**
- Google Drive 對外部網站顯示圖片偶爾會不穩，這是已知現象
- 點下方的「在 Drive 開啟」連結還是可以看到完整圖
- 若頻繁發生，未來可考慮把圖片改存 Cloudinary（免費）

**Q：想看老師的管理後台？**
- 你的 Google 試算表就是最強的後台：排序、篩選、匯出、備註都可以做
- 需要刪除某則產出，在試算表刪掉該列即可（注意：檔案還在 Drive 裡，要同步刪）
